Interactive machine learning for health informatics: when do we need the human-in-the-loop?

Abstract

    ML的目标是开发算法，可以学习和随着时间的推移而改进，并用于预测。大多数ML研究人员专注于 automatic machine learning（aML），例如，在语音识别、推荐系统、自动驾驶等。该方法极大地受益于大量训练数据。在样本数据较少的情况下，交互机器学习（interactive machine learning，iML）会更有用。
    我们将iML暂且定义为：algorithms that can interact with agents and can optimize their learning behavior through these interactions, where the agents can also be human.即human-in-the-loop，人类的专业知识可以通过启发式的样本选择来帮助减少指数搜索空间，大大降低了复杂性。

Introduction

    医疗数据可能包含缺失的数据、嘈杂的数据、肮脏的数据、不需要的数据，最重要的是，医疗领域的一些问题难以用自动机器学习算法来解决。自动机器学习算法的复杂性阻碍了非专家无法应用这类解决方案。因此，领域专家的知识的集成是必不可少的，而领域专家与数据的交互将极大地增强知识发现过程。协同两个领域：人机交互（HCI）和数据挖掘（KDD）
    iML：algorithms that can interact with both computational agents and human agents（oracles） and can optimize their learning behavior through these interactions.

From black-box to glass-box: where is the human-in-the-loop?

    上图中：
    1：输入数据；2：pre-processing；3：human agent(s) interacting with the computational agent(s)；4：final check done by the human expert
    A：无监督学习，不需要人手动标记数据，与人类的学习方式非常相似：当一个孩子在学习看东西时，它不会经常被告知正确的答案是什么，他们只是看，即使它的母亲在解释“那是雪”——实际上，这对孩子来说是很少的信息，人的大脑中有$10^{14}$神经元连接，但在每秒中，只有$10^{9}$在活动，因此，为了使剩下的$10^{5}$活动，计算速度没有很大作用，重要的是并行输入大量数据；人类专家可以在末端检查结果（1A），因此即使是自动机器学习算法也需要专家以参数调整形式的人类交互。
    B：监督学习，人类为训练数据提供标签，样本越多越好。
    C：半监督学习，混合了标记和未标记的数据，算法根据与给定数据的相似性度量找到预测标签。
    D：Human-in-the-loop，人类专家作为agent直接参与实际学习阶段，逐步影响结果。

Motivation for iML: when is the human-in-the-Loop beneficial?

    在iML中，正确的实验和评估不仅更加困难和耗时，而且难以复制，甚至不可能复制，因为人类代理是主观的、个体的，而aML更容易评估，可以更快地发表。
    医生通常可以做出非常可靠的诊断，但却不能明确地解释他们的规则。iML可以帮助算法具备这种“本能”的知识和学习。

Example: subspace clustering

    Local algorithms for interactive clustering：从数据的初始聚类开始，用户可以直接与算法逐步交互；在每个步骤中，用户以拆分和合并请求的形式对当前集群提供有限的反馈；该算法对与用户反馈一致的聚类进行局部编辑；这种编辑旨在改进循环中的人类所指出的集群中有问题的部分；该算法的目标是快速收敛（使用尽可能少的请求）到用户满意的聚类。

总结

    人类非常有能力从相对较少的样本中探索性地学习模式，而经典的监督ML需要大量的数据集和较长的处理时间。在生物医学领域，往往缺少大量的训练数据集，并且医生需要实时的结果，aML很难满足。
